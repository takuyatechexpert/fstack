# セキュリティ

## シークレット管理

- API キー・トークン・パスワードをコードに直接記述しない
- `.env` ファイルや環境変数経由で注入する
- シークレットを含むファイルは `.gitignore` に追加する
- コミット前に `git diff --cached` でシークレットの混入を確認する

## スキル開発時の認証情報ハンドリング

カスタムスキル（`~/.claude/skills/*`）で API トークン・API キー等を扱う際、
transcript ログへの平文露出を防ぐための原則。

### 優先順位（上ほど推奨）

1. **MCP サーバー経由** — サーバーが認証情報を保持し、Claude 本体には渡さない。最も安全
2. **言語サブプロセス経由** — スキル内 Python/Node スクリプトが `.env` を直読みし、HTTP クライアントで完結
3. **Bash `$VAR` 参照のみ** — `curl "...?key=$API_KEY"` のように **変数参照だけ** をコマンドに書く。値が transcript に焼き付かない

### 禁止事項

- **値の標準出力** — `echo $TOKEN` `printf $KEY` は値が transcript に残る
- **詳細トレース** — `curl -v` `--trace` `--trace-ascii` はリクエスト全文が露出
- **シェルトレース** — `set -x` `bash -x` は全変数展開を出力
- **環境ダンプ** — `printenv` `env`（単体実行）

### スキル作成時のチェックリスト

- [ ] 認証情報は `.env` に集約、ハードコード禁止
- [ ] 可能なら MCP サーバーまたは言語サブプロセスに閉じ込める
- [ ] Bash で使う場合、`$VAR` 参照のみ（値を echo/printf/cat しない）
- [ ] デバッグ時も `curl -v` / `set -x` を使わない（`curl -w "%{http_code}"` など必要情報のみ）
- [ ] エラーメッセージに変数値を含めない

### Hook での強制（任意）

`PreToolUse` hook で危険パターン（`echo $*TOKEN`, `curl -v`, `set -x` 等）をブロック可能。
fstack はオプションのセキュリティ hook を `hooks/` 配下で配布している。
利用者は `~/.claude/settings.json` で opt-in できる（詳細は `hooks/README.md`）。

## 入力バリデーション

- ユーザー入力は必ずバリデーションする（型・長さ・形式）
- SQL クエリにはプレースホルダ（プリペアドステートメント）を使用する
- HTML 出力時はエスケープ処理を行う（XSS 防止）
- ファイルパスの入力にはディレクトリトラバーサル対策を行う

## エラーハンドリング

- エラーメッセージにスタックトレース・内部パス・DB スキーマを含めない
- ユーザー向けエラーと開発者向けログを分離する
- 認証エラーでは「ユーザー名が間違っています」等の詳細を返さない（情報漏洩防止）

## 認証・認可

- 認証トークンの有効期限を設定する
- 権限チェックはコントローラ層で必ず実施する
- CSRF トークンを適用する（状態変更リクエスト）

## 依存関係

- 既知の脆弱性を持つパッケージを使用しない
- `npm audit` / `composer audit` 等で定期的に確認する

## OWASP Top 10 チェックリスト

コードレビュー・実装時に以下を意識する：

1. **インジェクション** — SQL / コマンド / LDAP インジェクション対策
2. **認証の不備** — セッション管理・パスワードポリシー
3. **機密データの露出** — 暗号化・転送時の保護
4. **XXE** — XML パーサーの外部エンティティ無効化
5. **アクセス制御の不備** — 水平・垂直権限昇格の防止
6. **セキュリティ設定ミス** — デフォルト設定の見直し
7. **XSS** — 出力エスケープ・CSP ヘッダー
8. **安全でないデシリアライゼーション** — 信頼できないデータの検証
9. **脆弱性のあるコンポーネント** — 依存関係の更新
10. **不十分なログ・監視** — セキュリティイベントの記録
