# ファイル操作

このルールはコアエンジニア向け。`rm` の挙動・glob 展開・git 管理の基本を理解している前提で、
**復元不可能な操作を不用意に行わない** ための原則のみ定義する。

## 破壊的操作の取扱い

ファイル・ディレクトリの削除や上書きは復元不可能。以下を徹底する:

### 削除の代替案

- **`git` 管理下** — `git rm` で履歴に残す
- **ローカル一時ファイル** — システムのゴミ箱（macOS Trash, Linux `~/.local/share/Trash` 等）への移動を推奨
- **プロジェクト固有のゴミ箱戦略** — チーム・利用者で別途運用しているなら、それに従う

### 直接の `rm -rf` を避ける

- 削除対象を `mv` で別ディレクトリに退避してから様子を見る
- 復元できることを確認してから物理削除

### スクリプト・Makefile 内の `rm`

- 既存スクリプト内の `rm` を変更する場合、ユーザーに確認を取る
- ビルド成果物のクリーンアップは慣習として OK（`make clean` など）

## 確認ルール

- 削除対象が想定通りか `ls` で事前確認
- glob 展開（`rm *.tmp` 等）は意図しないファイルに当たらないか確認
- ホームディレクトリ・システムディレクトリ配下では特に慎重に

## カスタマイズ

このルールはあくまで原則。プロジェクトや個人ワークフローに応じて拡張可能:

- 「削除は必ず特定のディレクトリへ `mv`」など独自ルールを上書き
- プロジェクト直下に `.claude/rules/file-operations.md` を置けば fstack 版を上書きできる
