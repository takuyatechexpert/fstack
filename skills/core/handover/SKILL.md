# Handover（セッション引き継ぎ）

コンテキスト劣化（Context Rot）を防ぐためのセッション引き継ぎスキル。
現セッションの作業状況を要約し、次セッションが即座に作業を継続できるドキュメントを生成する。

## 保存先

`~/.claude/handovers/{project-name}/{YYYY-MM-DD}_{summary}.md`

project-name は cwd のベースディレクトリ名から取得する。

## ワークフロー

### Phase 1: 情報収集

現在のセッションから以下を整理する:

1. **プロジェクト・ディレクトリ**: `pwd`
2. **Git の状態**: `git status --short`, `git diff --stat`, `git log --oneline -5`
3. **今回のセッションで触ったファイル**: git diff と会話履歴から判断
4. **関連する Issue / PR 番号**: 会話履歴から抽出（あれば）

### Phase 2: ユーザーとの対話

収集した情報をもとに、以下を提示して確認する:

```
引き継ぎドキュメントを作成します。

- タスク概要: {推定}
- Issue/PR: {あれば}
- 完了した作業: {会話から推定}
- 残タスク: {推定}
- 重要な判断: {推定}

修正はありますか？（問題なければそのまま作成します）
```

### Phase 3: ドキュメント生成

以下のテンプレートで生成する:

```markdown
# セッション引き継ぎ: {タスク概要}

**作成日時**: {YYYY-MM-DD HH:MM}
**プロジェクト**: {project-name}
**ブランチ**: {current-branch}

## タスク識別子
- Issue: #{issue-number}（あれば）
- PR: #{pr-number}（あれば）

## 完了した作業
- {具体的に「どのファイルのどの関数まで」など}

## 残タスク
- [ ] {未完了の作業}

## 次のステップ
1. {次セッションで最初にやるべきこと}

## 重要な判断・コンテキスト
- {このセッションで下した判断とその理由}
- {次セッションが知っておくべき背景}

## 関連ファイル
- {path/to/file} - {変更内容の要約}
```

### Phase 4: 保存と案内

1. `mkdir -p ~/.claude/handovers/{project-name}/` でディレクトリ作成
2. ドキュメントを保存
3. 次セッションでの再開方法を案内:

```
保存しました: ~/.claude/handovers/{project-name}/{filename}.md

次セッションでは以下のプロンプトで再開してください:
「前回の引き継ぎを読んで作業を続けてください: ~/.claude/handovers/{project-name}/{filename}.md」
```

## 注意事項

- 引き継ぎドキュメントの生成自体がコンテキストを消費するため、簡潔に保つ
- ユーザーの確認を得てから保存する
- secret / 機密情報を含めない（.env の値、APIキー等）
- 1セッションで複数回実行可能（タスク切り替え時など）
